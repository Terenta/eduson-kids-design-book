# Дизайн-бук презентаций Eduson Kids

Единый визуальный язык слайдов курса программирования для детей: цвет, типографика, сетка 1280 × 720, декор «Орбита», компоненты, живые шаблоны слайдов и интерактивные блоки для вопросов.

**Смотреть:** открой [`index.html`](index.html) в браузере — файл самодостаточен, внешних библиотек нет (только Google Fonts, с системными фолбэками).

## Что внутри

| Раздел | Содержимое |
|---|---|
| 01 Принципы | 6 правил стиля («экран — это окно», «один акцент на слайд»…) |
| 02 Цвет | 16 токенов с копированием HEX: фиолет `#8C7FF6`, неон-зелень `#3DF79D`, чернила `#17171C` |
| 03 Типографика | Inter Tight / Inter / JetBrains Mono, шкала слайда |
| 04 Сетка | Зоны слайда 1280 × 720, поля 80 px |
| 05 Декор «Орбита» | Планеты (кубок, сердце, звезда, гем, буквы X/Y/A/B) и правила расстановки |
| 06 Компоненты | Бейджи, карточка-окно, код-блок, спич-бабл, кнопки, варианты ответа |
| 07 Шаблоны слайдов | 7 живых макетов в масштабе (титул, код, разделитель, вопросы, тезисы, практика, интерактив) |
| 08 Интерактив | 8 рабочих шаблонов: квиз, мульти-выбор, правда/ложь, «что выведет код?», пропуски, флеш-карты, пары, открытый вопрос |
| 09 Можно / нельзя | 6 пар типичных нарушений |
| 10 Ассеты | SVG-фоны и иконки с копированием кода |

## Структура

```
index.html            ← собранный дизайн-бук (НЕ править руками)
src/
  parts/              ← HTML-части разделов (p00…p12)
  components.json     ← интерактивные блоки (html/css/js каждого шаблона)
  codex-svg/          ← SVG-фоны и иконки
  component_contract.md  ← контракт интерактивного компонента
  svg_spec.md         ← спека SVG-ассетов
  assemble.py         ← сборщик
```

## Пересборка после правок

Правь исходники в `src/`, затем:

```bash
cd src
PYTHONUTF8=1 python assemble.py   # соберёт ../index.html
```

Сборщик инлайнит SVG, вставляет интерактивные блоки из `components.json` и проверяет, что в файле не осталось незаполненных маркеров.

## Дизайн-токены

Все цвета и шрифты — CSS-переменные `--ek-*` в `src/parts/p00-head.html` (объявлены в `:root`). Интерактивные блоки используют только их — тема меняется в одном месте.
