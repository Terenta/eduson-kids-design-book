Идея урока

Что значит «страница интерактивна»

Интерактивная страница умеет отвечать на действия пользователя: клик по кнопке, ввод текста, отметку чекбокса, прокрутку, наведение мыши.

Кнопка с hover-эффектом из урока 2 — это ещё не интерактивность. Настоящая интерактивность — когда после действия пользователя меняется содержимое страницы: появляется новый элемент, обновляется текст, сохраняются данные.

Как это устроено

Три шага интерактивности

1 · Действие

Пользователь нажимает кнопку, печатает текст, отмечает чекбокс.

2 · Событие

Браузер замечает действие и отправляет «событие» (event) в JavaScript.

3 · Реакция

Скрипт получает событие и меняет страницу: добавляет элемент, обновляет текст, сохраняет данные.

Сегодня учимся ловить эти события и менять страницу в ответ — через диалог с DeepSeek.
Карта страницы

Что такое DOM

DOM (Document Object Model) — это модель страницы в памяти браузера. Каждый тег HTML превращается в узел, к которому можно обратиться из JavaScript.

HTML — это текст

То, что вы пишете в файле index.html.

DOM — это объекты

То, во что браузер превращает HTML, чтобы JS мог это менять «на лету».

DOM — это дерево

Страница как дерево элементов

HTML
<body> <h1>Мои задачи</h1> <input id="field"> <button id="add">Добавить</button> <ul id="list"> <li>Купить хлеб</li> </ul> </body>
ДЕРЕВО DOM
body ├── h1 "Мои задачи" ├── input#field ├── button#add "Добавить" └── ul#list └── li "Купить хлеб"

JS умеет дойти до любого узла и изменить его.

Шаг 1 работы с DOM

Как JavaScript находит элемент

ПО ID
// HTML: <button id="add"> const btn = document.getElementById('add'); // Современный универсальный способ const btn2 = document.querySelector('#add');
ПО КЛАССУ / ТЕГУ
// Первый элемент с классом .task const t = document.querySelector('.task'); // ВСЕ элементы <li> — список const all = document.querySelectorAll('li');
Правило: querySelector возвращает один элемент (первый совпавший). querySelectorAll — список всех совпавших.
Шаг 2 работы с DOM

Как JavaScript меняет элемент

JAVASCRIPT
// 1. Поменять текст document.querySelector('h1').textContent = 'Привет!'; // 2. Добавить / убрать CSS-класс element.classList.add('done'); element.classList.remove('done'); element.classList.toggle('done'); // есть → убрать, нет → добавить // 3. Создать новый элемент и вставить const li = document.createElement('li'); li.textContent = 'Новая задача'; document.querySelector('ul').appendChild(li); // 4. Удалить элемент li.remove();

Этих 4 операций хватит, чтобы собрать список задач целиком.

Сигналы от пользователя

Что такое событие

Событие (event) — это сигнал от браузера: что-то произошло. Клик, ввод символа, отметка чекбокса, нажатие клавиши. JavaScript не «следит» постоянно — он подписывается на нужные события и ждёт.

Аналогия: вы не стоите весь день у двери. Вы вешаете звонок. Когда придёт гость — звонок сработает. addEventListener — это и есть «повесить звонок».
Справочник

Главные виды событий

click
Клик мышью по элементу. Кнопки, ссылки, любые блоки.
input
Пользователь печатает в поле ввода. Срабатывает на каждый символ.
change
Значение изменилось и поле потеряло фокус. Чекбоксы, выпадающие списки.
submit
Отправка формы — нажатие Enter или кнопки внутри формы.
keydown
Нажата клавиша на клавиатуре. Пригодится на уроке 4 для игры.
mouseover
Курсор навёлся на элемент. Часто заменяют CSS-эффектом :hover.

Полезное знание

Что такое localStorage

localStorage — это маленькое хранилище в браузере. Данные не исчезают при перезагрузке и закрытии вкладки.

  • До ~5 МБ на один сайт
  • Хранит только строки (ключ → значение)
  • Привязан к домену: другой сайт не увидит ваши данные
JAVASCRIPT
// Сохранить localStorage.setItem('tasks', '...'); // Прочитать const saved = localStorage.getItem('tasks'); // Удалить localStorage.removeItem('tasks');
Ключевой приём

Массив → строка → массив

localStorage хранит только строки. Массив задач превращаем в строку и обратно:

JAVASCRIPT
const tasks = [ { text: 'Купить хлеб', done: false }, { text: 'Сделать ДЗ', done: true }, ]; // Сохранить: массив → строка JSON localStorage.setItem('tasks', JSON.stringify(tasks)); // Прочитать: строка JSON → массив const saved = localStorage.getItem('tasks'); const restored = saved ? JSON.parse(saved) : [];
stringify — «упаковать» в строку. parse — «распаковать» обратно. Без них в localStorage попадёт [object Object].
Правка 5 · анимация

Плавное появление и удаление

→ В тот же чат
Сделай анимации: • Новая задача плавно появляется (opacity 0 → 1, выезжает на 8px снизу) за 200мс • При удалении задача плавно исчезает (opacity 1 → 0, scale 1 → 0.95) за 200мс, и только потом удаляется из DOM Используй CSS-переходы. Не трогай логику добавления/удаления — только добавь классы и переходы. Изменения в style.css и небольшие правки в script.js. Верни изменённые блоки.
Полезное знание

Как работают CSS-переходы

transition плавно анимирует изменение свойства между двумя состояниями.

CSS
.task { opacity: 1; transform: translateY(0); transition: opacity 200ms, transform 200ms; /* что и за сколько */ } /* класс, который добавляет JS при удалении */ .task.removing { opacity: 0; transform: scale(0.95); }
JS только добавляет/убирает класс — всю плавность делает CSS. Это быстро и не нагружает страницу.
Проверка · вопрос 1

Изучите ситуацию и ответьте

Вопрос 1

Вы добавили в localStorage 5 задач. Открыли тот же сайт в режиме «инкогнито» — задач там нет. Почему?

Режим «инкогнито» использует отдельное хранилище. Когда вы закрываете окно инкогнито, всё содержимое localStorage очищается. Так браузер защищает приватность.

В обычной вкладке localStorage сохраняется, пока пользователь сам его не очистит.

Проверка · вопрос 2

Изучите ситуацию и ответьте

Вопрос 2

Кнопка «Добавить» не реагирует на клик. В консоли — ошибка Cannot read properties of null на строке addBtn.addEventListener(...). В чём причина?

Скрипт подключён в <head> и выполняется до того, как браузер построил кнопку. getElementById('add') вернул null — кнопки ещё нет.

Решение: перенести <script src="script.js"></script> в самый конец <body>.
