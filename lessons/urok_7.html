Новый формат

Что такое JSON

JSON — текстовый формат для структурированных данных. Выглядит почти как словари и списки Python, которые вы видели в игре.

JSON
{ "date": "2026-06-12", "text": "Выиграл турнир по шахматам", "tags": ["шахматы", "победа"] }
Фигурные скобки { } — как словарь (поле: значение), квадратные [ ] — как список. JSON читается и человеком, и программой.

Как это работает

JSON похож на структуры Python

JSON нужен, чтобы хранить структуру в обычном текстовом файле. Он похож на знакомые вам список и словарь.

Типы совпадают

Словарь { } ↔ object, список [ ] ↔ array, строка ↔ string, число ↔ number, True/False ↔ true/false.

Две команды

В файл — json.dump и json.load (ими и будем работать). А dumps/loads — то же самое, но со строкой в памяти.

Один параметр обязателен для русского: ensure_ascii=False. Без него слово «шахматы» превратится в коды вроде ш....

Запись сохраняется между запусками

→ В тот же чат
Доработай diary.py, остальное не меняй. Пункт «новая запись»: - Спроси текст записи через input - Добавь текущую дату из datetime - Собери запись как словарь с полями дата и текст и добавь её в список записей - Сохрани весь список в файл diary.json через json.dump с ensure_ascii=False и indent=2 - При запуске программы, если diary.json уже есть, загрузи записи из него через json.load Верни весь файл целиком.
Разбор кода

Как список попадает в файл и обратно

PYTHON
with open("diary.json", "w", encoding="utf-8") as f: json.dump(entries, f, ensure_ascii=False, indent=2) # сохранили весь список ... with open("diary.json", "r", encoding="utf-8") as f: entries = json.load(f) # прочитали весь список
json.dump сохраняет весь список целиком, json.load читает его обратно. ensure_ascii=False — русские буквы остаются читаемыми, indent=2 — файл с отступами, его видно глазами.

Прочитай запись и верни 3-5 тегов через запятую, одной строкой, без пояснений». Ответ нейросети разбей на список тегов и сохрани в поле tags записи. Ключ бери из API_KEY. Верни весь файл.
Разбор кода

Из строки нейросети — в список тегов

PYTHON
headers = {"Authorization": f"Bearer {API_KEY}"} data = { "model": "deepseek-chat", "messages": [ {"role": "system", "content": "Ты помощник дневника, верни теги через запятую"}, {"role": "user", "content": text}, ], } response = requests.post(url, headers=headers, json=data) answer = response.json()["choices"][0]["message"]["content"] tags = [t.strip() for t in answer.split(",")]
Нейросеть вернула строку "шахматы, победа, дружба". split(",") режет её по запятым в список, strip() убирает лишние пробелы. Пишете запись — теги добавляются автоматически: это видимый результат работы нейросети.
Полезное знание

Два параметра для читаемого файла

Без ensure_ascii=False

В файле вместо слова — коды: "шах...".

Программа прочитает, а человек — нет.

С ensure_ascii=False, indent=2

В файле читаемо и с отступами: "шахматы".

Дневник можно открыть и прочитать как текст.

С этими параметрами JSON-файл читают и человек, и программа.

Разбор кода

Почему JSON нельзя дописывать

Новая запись добавляется в дневник в три шага — прочитать, добавить, сохранить целиком:

PYTHON
entries = load_entries() # прочитали весь список из файла entries.append(new_entry) # добавили запись в список save_entries(entries) # записали весь список заново
JSON нельзя открыть в режиме 'a' и дописать словарь в конец: в файле окажется два JSON подряд, и прочитать его уже не получится. Это отличие от history.txt на уроке 5, где режим 'a' работал.
Проверка · вопрос 1

Изучите ситуацию и ответьте

Вопрос 1

Дима решил ускорить сохранение и открыл diary.json в режиме 'a', дописав новый словарь в конец. Программа сохранила без ошибок, но при следующем чтении упала с JSONDecodeError. Почему?

В файле оказалось два JSON подряд — старый список и приписанный словарь. Это уже не валидный JSON, и json.load не может его прочитать.

Правильно: прочитать весь список через json.load, добавить запись в список, сохранить весь список через json.dump.

Проверка · вопрос 2

Изучите ситуацию и ответьте

Вопрос 2

Настя записала «Выиграл турнир по шахматам», открыла diary.json, а там вместо слова «шахматы» — набор кодов вида шах.... Программа работает, но файл нечитаем. Что она забыла?

Параметр ensure_ascii=False в json.dump. Без него русские буквы сохраняются как коды \uXXXX.

С ensure_ascii=False в файле остаётся читаемый русский текст.

Отладка

Читаем ошибки JSON

ТЕРМИНАЛ
Traceback (most recent call last): File "diary.py", line 8, in load_entries return json.load(f) json.decoder.JSONDecodeError: Expecting value

Снизу вверх: последняя строка — что случилось, строкой выше — где. JSONDecodeError значит: файл пустой или повреждён.

Если застряли

  • Скопируйте traceback целиком в чат DeepSeek — попросите объяснить и починить.
  • Проверьте ensure_ascii=False и режим 'w' при сохранении.
  • Откройте diary.json глазами — валиден ли он.

Тот же подход, что console.log в «Змейке» и print-метки на уроке 5.
