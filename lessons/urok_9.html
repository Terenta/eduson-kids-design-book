До сих пор программы работали в терминале — их видел только автор. Прогноз погоды, дневник и советчик работали локально, но ими было трудно поделиться.

Вспоминаем

Что такое бот и Bot API

Бот — это обычная программа, которая общается с людьми в мессенджере: получает их сообщения и отправляет ответы. За ботом всегда стоит код — сегодня это будет ваш bot.py.

Bot API

Telegram даёт Bot API — набор правил, по которым программа получает сообщения и отправляет ответы. Обращаться к нему напрямую сложно.

Как это работает

Три звена и long polling

Между вами и ботом всегда три звена: пользователь пишет в Telegram, а ваш bot.py забирает сообщение и отправляет ответ обратно.

1 · Пользователь

Пишет боту в Telegram со своего телефона.

2 · Telegram

Хранит сообщение и ждёт, пока программа его заберёт.

3 · Ваш bot.py

Забирает сообщение, готовит ответ и шлёт назад.

Long polling: бот сам постоянно спрашивает у Telegram «есть новые сообщения?». Это делает одна строчка — run_polling(). Пока она работает — бот в сети.

Запуск и разбор

Бот-эхо отвечает в Telegram

PYTHON
import os from telegram import Update from telegram.ext import (Application, MessageHandler, filters, ContextTypes) TOKEN = "123456:ABC-DEF..." # пока прямо здесь async def on_message(update: Update, context: ContextTypes.DEFAULT_TYPE): text = update.message.text await update.message.reply_text(text) app = Application.builder().token(TOKEN).build() app.add_handler(MessageHandler( filters.TEXT & ~filters.COMMAND, on_message)) app.run_polling() # бот спрашивает Telegram
async def
Обработчик асинхронный, перед reply_text — await.
filters.TEXT & ~COMMAND
Ловим обычный текст, но не команды.
run_polling()
Бесконечный цикл: бот сам спрашивает про сообщения.
Бот повторил ваше сообщение в Telegram? Каркас готов — дальше подключим ответы через нейросеть.

Если бот-эхо не запустился

Регистрация бота

Заводим бота у @BotFather

Бота создаёт Telegram через официального бота @BotFather. Пять коротких шагов — и у вас есть токен.

  1. Найдите @BotFather — в поиске Telegram, официальный, с синей галочкой. Откройте чат.
  2. Отправьте /newbot — команда запускает создание нового бота.
  3. Имя бота — как он подписан в чате, любое, например «Мой AI-помощник».
  4. @username — заканчивается на bot, например my_ai_helper_bot.
Шаг 5: @BotFather присылает токен вида 123456:ABC-DEF1234... — ключ от вашего бота. На первом запуске положим его в код, а затем перенесём в .env.
Безопасность

Токен — это пароль от бота

Токен даёт полный контроль над ботом. Кто получил токен — управляет ботом от вашего имени: читает сообщения, отвечает, меняет настройки.

Никогда

Не выкладывайте токен в общий чат, не пишите прямо в коде на скриншотах, не отправляйте в мессенджере. На скриншотах для домашки — замазывать.

Если утёк

Немедленно перевыпустите у @BotFather командой /revoke или /token. Старый токен сразу перестанет работать.

Поэтому токен хранят так же, как ключ DeepSeek на уроке 8 — в файле .env, который не попадает в репозиторий.

Момент урока · мостик

От эха — к нейросети за пять правок

Сейчас бот-эхо пока повторяет ваше сообщение. Ядро уже собрано: MessageHandler ловит текст, run_polling держит бота в сети. Осталось поменять одну строчку — что бот отвечает.

PYTHON
async def on_message(update: Update, context: ContextTypes.DEFAULT_TYPE): text = update.message.text answer = ask_deepseek(text) # раньше здесь было text — теперь ответ DeepSeek await update.message.reply_text(answer)
Каждая из пяти правок — отдельный запрос к DeepSeek в тот же чат, и после каждой бот в Telegram меняет поведение: отвечает нейросетью, знает команды, получает характер и память.

Правка 1 · токен в .env

Токен переносим в .env

Секрет уходит из кода в отдельный файл — как ключ DeepSeek на уроке 8. В коде читаем токен через python-dotenv.

→ В тот же чат
Заведи бота у @BotFather и получи токен. Положи токен в файл .env строкой BOT_TOKEN=..., а в коде читай через python-dotenv. Проверь, что бот-эхо отвечает в Telegram. Верни весь файл.
Разбор кода

Как токен попадает в код из .env

PYTHON
from dotenv import load_dotenv import os load_dotenv() # прочитали файл .env рядом с bot.py TOKEN = os.getenv("BOT_TOKEN") # достали токен по имени

load_dotenv() читает файл .env, который лежит рядом с bot.py. os.getenv("BOT_TOKEN") достаёт значение по имени — ровно та же привычка, что с ключом DeepSeek на уроке 8. Токена в самом коде больше нет.

Правка 2 · отвечает нейросеть

Бот отвечает через нейросеть

Главный шаг урока: на каждое сообщение бот отправляет текст в DeepSeek и возвращает ответ нейросети — эхо превращается в AI-собеседника.

→ В тот же чат
Подключи DeepSeek: на любое сообщение бот отправляет его текст в нейросеть и отвечает её ответом. Ключ DeepSeek положи в .env строкой DEEPSEEK_KEY=... и читай через python-dotenv. Верни весь файл.
Разбор кода

Путь сообщения: Telegram → DeepSeek → назад

PYTHON
async def on_message(update: Update, context: ContextTypes.DEFAULT_TYPE): text = update.message.text # что написал человек answer = ask_deepseek(text) # та же функция, что на уроках 5-8 await update.message.reply_text(answer) # ответ нейросети — в чат
Три шага: берём текст из update.message.text, отдаём его в ask_deepseek(text) — ту самую функцию с уроков 5-8, её не переписываем, — и возвращаем ответ через reply_text. Пишете боту любую фразу — он отвечает по смыслу, а не повтором.
Полезное знание

Два слова: async и await

Обычная функция

Бот занят одним человеком — остальные ждут в очереди, пока придёт ответ.

Один медленный запрос блокирует остальных.

async def и await

Пока бот ждёт ответ DeepSeek для одного — он принимает сообщения от других.

await перед reply_text обязателен.

Обработчик — это async def, а перед отправкой ответа стоит await. Так бот не блокируется на одном пользователе и остаётся доступным для всех.

Правка 3 · /start и /help

Первое, что видит новый собеседник

Команды со слэшем — то, с чего человек начинает знакомство с ботом. Их ловит отдельный обработчик.

→ В тот же чат
Добавь команды: /start — приветствие «Привет! Я бот на нейросети, спроси что угодно», /help — короткая подсказка. Используй CommandHandler. Верни весь файл.
Разбор кода

Команды идут отдельно от обычного текста

PYTHON
async def start(update: Update, context: ContextTypes.DEFAULT_TYPE): await update.message.reply_text("Привет! Я бот на нейросети, спроси что угодно") app.add_handler(CommandHandler("start", start)) app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, on_message))

Команды со слэшем (/start, /help) ловит CommandHandler, а обычный текст — MessageHandler. Вот зачем в фильтре стоит ~filters.COMMAND: чтобы /start не улетел в нейросеть, а попал в свой обработчик.

Разбор кода

Характер живёт в системной роли

PYTHON
messages = [ {"role": "system", "content": "Ты дружелюбный помощник-подросток"}, {"role": "user", "content": text}, ]
Характер задаёт первое сообщение с ролью system — ровно как на уроках 6-7. Меняете одну строку роли — и на один и тот же вопрос бот отвечает другим тоном. Сам вопрос при этом не трогаем.
Правка 5 · память и «печатает…»

Память и «печатает…»

Финальная правка собирает бота в законченный вид: он помнит контекст разговора и показывает статус, пока думает над ответом.

→ В тот же чат
Финальная правка. Пусть бот помнит несколько последних сообщений каждого пользователя: заведи словарь по user_id со списком messages и передавай эту историю в DeepSeek, чтобы он понимал контекст разговора. И пока бот думает над ответом, пусть показывает статус «печатает…» через send_chat_action. Верни весь файл.
Разбор кода

Почему память — словарь по user_id

Бот один, а собеседников много. У каждого своя история, поэтому храним её в словаре по user_id — список сообщений с ролями, тот самый формат, что уходит в DeepSeek на уроках 5-8.

PYTHON
history = {} # user_id -> список сообщений с ролями async def on_message(update: Update, context: ContextTypes.DEFAULT_TYPE): user_id = update.message.from_user.id msgs = history.setdefault(user_id, []) # своя история для каждого msgs.append({"role": "user", "content": text}) # запомнили вопрос answer = ask_deepseek(msgs) # весь список — в DeepSeek msgs.append({"role": "assistant", "content": answer}) # и ответ бота
Одна общая переменная не подошла бы: сообщения Вани и Насти смешались бы. Ключ словаря user_id разделяет истории — у каждого свой контекст.

Вопрос 1

Ваня написал бота, запустил, но окно программы не закрывается и не возвращает приглашение терминала. Он думает, что программа не работает. Так ли это?

Нет. run_polling() запускает бесконечный цикл long polling: бот сам постоянно спрашивает у Telegram «есть новые сообщения?». Программа работает не закрываясь — пока она запущена, бот в сети.

Остановить бота — Ctrl+C в терминале.

Вопрос 2

Катя случайно выложила токен бота в чат класса. Что теперь может любой из одноклассников и что делать?

По токену любой может полностью управлять ботом от её имени — читать сообщения и отвечать за неё.

Токен нужно немедленно перевыпустить у @BotFather (/revoke или /token) — старый сразу перестанет работать. Хранить токен только в .env.

Правила работы с ботом

01 · Интерфейс в мессенджере
Программа отвечает людям прямо в Telegram, а не только в локальном терминале.
02 · run_polling — это норма
Программа работает и не закрывается — значит бот в сети. Это штатная работа. Остановить — Ctrl+C.
03 · Токен — это пароль
Не публикуем, на скриншотах скрываем, храним только в .env. Утёк — перевыпускаем у @BotFather.
04 · Обработчик — async и await
Ловим сообщения через async def, а перед reply_text ставим await — тогда бот не блокируется на одном пользователе.
05 · Характер и память
Характер задаём системной ролью, а память — словарём по user_id: у каждого свой разговор.
