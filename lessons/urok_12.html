Идея урока

Собеседник, который печатает вживую и помнит

На уроке 11 writer.py выдавал ответ целиком после паузы — а в ChatGPT текст появляется постепенно, и у собеседника есть характер.

Сегодня две новинки: включаем стриминг — ответ печатается фрагментами через stream=True, — и задаём собеседнику личность системным промптом. А чтобы он помнил разговор, каждый его ответ дописываем в диалог как роль assistant.

Управлять чатом будем командами, как в чат-ботах: /start — начать заново, /help — список команд, /clear — стереть память диалога.

Роли system и user знакомы с урока 5 — новое здесь только assistant (память) и stream=True (живая печать).

Вспоминаем

Как отвечает ChatGPT

На уроке 11 мы вызывали client.chat.completions.create(...) и ждали весь ответ целиком: пауза — и текст появлялся сразу весь.

PYTHON
response = client.chat.completions.create( model="deepseek-chat", messages=messages, ) print(response.choices[0].message.content) # весь текст сразу, после паузы
А в ChatGPT текст появляется постепенно — не нужно ждать конца. Тот же приём мы включим у себя одним параметром.
Новый приём

Что такое стриминг

Без стриминга

Модель дописывает весь ответ, и лишь потом он появляется на экране целиком. Долгий ответ — долгая пауза с пустым экраном.

Так работал writer.py.

Стриминг (stream=True)

Ответ приходит маленькими кусочками — chunks. Каждый кусочек печатаем сразу — текст появляется постепенно, как в ChatGPT.

Это включим сегодня.

Один параметр stream=True меняет способ доставки: вместо целого ответа — поток кусочков, которые печатаются вживую.

Вспоминаем и добавляем

Роли и характер собеседника

Диалог с моделью — это список messages, где у каждого сообщения есть роль. Роли system и user вы знаете с урока 5.

PYTHON
messages = [ {"role": "system", "content": "Ты дружелюбный собеседник по имени Макс, любишь космос."}, ] messages.append({"role": "user", "content": user_text})

system

Личность собеседника: имя, стиль речи, интересы. Задаёт характер всех ответов.

user

Реплики человека — то, что печатаете вы.
assistant — новое: ответы нейросети. Дописываем их в messages — так рождается память.

Если warmup.py не запустился

Как это работает

Стрим — это цикл по кусочкам

При stream=True ответ приходит не целиком, а маленькими кусочками (chunks). Мы идём по ним циклом и печатаем каждый сразу — текст «печатается» на глазах.

PYTHON
answer = "" for chunk in stream: piece = chunk.choices[0].delta.content or "" print(piece, end="", flush=True) # печатаем кусочек сразу answer += piece # и копим весь ответ print()
delta.content — очередной кусочек текста. Пишем or "", потому что иногда кусочек пустой. Всё собранное складываем в answer — пригодится для памяти.
Как это работает

Память диалога — это список messages

Собеседник помнит разговор, потому что весь диалог лежит в списке messages. Роли system и user знакомы с урока 5 — сегодня добавляем assistant.

PYTHON
messages = [ {"role": "system", "content": "Ты дружелюбный собеседник по имени Макс, любишь космос."}, ] messages.append({"role": "user", "content": user_text}) # ... получили ответ ... messages.append({"role": "assistant", "content": answer}) # память диалога
Главное правило: каждый ответ дописываем в messages как assistant. Забыли — и модель видит только последний вопрос, отвечает без предыдущего контекста.
Как это работает

Личность задаёт системный промпт

Первое сообщение с ролью system — это характер собеседника. Оно управляет тоном всех ответов сразу, как режиссёр задаёт роль актёру.

Коротко — нейтрально

«Ты собеседник» — отвечает нейтрально, без выраженного характера, как обычный бот.

Детально — ярко

«Ты дружелюбный собеседник по имени Макс, любишь космос» — задаёт имя, стиль речи и интересы. Собеседник держится в образе.

Чем детальнее system (имя, манера, увлечения) — тем выразительнее собеседник. Личность меняется без единой правки кода — только текстом промпта. Сегодня заведём четыре: учитель, друг, пират, философ.

Правка 1 · стриминг

Ответ, который печатается на глазах

Сейчас ответ приходит целиком после паузы. Включаем стриминг: теперь он приходит кусочками и печатается по фрагментам — как в ChatGPT.

→ В тот же чат
Включи в companion.py стриминг вместо обычного ответа: добавь stream=True в запрос и печатай ответ по словам циклом по chunk.choices[0].delta.content с print(end="", flush=True). Верни весь файл.
Разбор кода

Как ответ появляется постепенно

PYTHON
stream = client.chat.completions.create( model="deepseek-chat", messages=messages, stream=True, # ответ придёт кусочками ) for chunk in stream: piece = chunk.choices[0].delta.content or "" print(piece, end="", flush=True) # печатаем по словам, без переносов print()

С stream=True ответ приходит не целиком, а кусочками (chunks). Цикл for chunk in stream забирает очередной кусочек через delta.content, а or "" подставляет пустую строку, если кусочек пустой. print(..., end="", flush=True) печатает их в одну строку и сразу — текст появляется на экране постепенно.

Разбор кода

Системный промпт задаёт характер

PYTHON
messages = [ {"role": "system", "content": "Ты дружелюбный собеседник по имени Макс, любишь космос."}, ] messages.append({"role": "user", "content": user_text})
Роль system — это первое сообщение в messages, и оно управляет тоном всех ответов. Роли system и user вы уже видели на уроке 5. Чем детальнее описание — имя, стиль речи, интересы — тем ярче звучит характер собеседника.
Полезное знание

Два параметра, которые делают живую печать

Обычный print

Каждый кусочек — с новой строки, текст копится и вылетает пачкой после паузы.

Фрагменты падают в столбик, постепенной печати нет

print(end="", flush=True)

end="" — без переноса, кусочки идут в одну строку. flush=True — показать сразу, не копить в буфере.

Текст появляется постепенно, как в ChatGPT

Два параметра — и стрим печатается вживую: в одну строку и без задержки.

Правка 3 · память диалога

Собеседник, который помнит разговор

Диалог по кругу: каждый ответ нейросети дописываем в messages — и собеседник помнит всё, о чём говорили раньше.

→ В тот же чат
Сделай диалог по кругу: копи сообщения в списке messages, добавляя каждый ответ нейросети как role assistant, — собеседник должен помнить, о чём говорили раньше. Выход по слову «выход». Верни весь файл.
Разбор кода

Роль assistant хранит контекст

PYTHON
answer = "" for chunk in stream: piece = chunk.choices[0].delta.content or "" print(piece, end="", flush=True) answer += piece # собираем ответ целиком messages.append({"role": "assistant", "content": answer}) # память диалога
Пока стрим печатается, мы копим кусочки в answer, а потом дописываем весь ответ в messages как role: assistant. Так модель на следующем шаге видит весь диалог целиком и помнит контекст. Забыли строку с assistant — собеседник отвечает без предыдущего контекста.

Разбор кода

Один выбор — свой системный промпт

PYTHON
persons = { "1": "Ты строгий, но добрый учитель. Терпеливо объясняешь и приводишь примеры.", "2": "Ты весёлый друг. Болтаешь просто и всегда поддерживаешь.", "3": "Ты старый пират. Подсказки для нейросети: - /start — начать заново: снова покажи выбор личности и очисти память диалога - /help — выведи список всех команд с пояснениями - /clear — сотри память диалога (список messages), но оставь выбранную личность - /exit — выход из программы вместо слова «выход» - команды проверяй до отправки запроса нейросети - имя собеседника печатай перед каждым ответом ANSI-цветом (например, \033[35m), без сторонних библиотек
Разбор кода

Команды перехватываем до запроса

Слэш-команды проверяем сразу после input — до обращения к нейросети. Команды /clear и /start меняют список messages, а /help печатает подсказку:

PYTHON
text = input("Вы: ") if text == "/help": print("Команды: /start /help /clear /exit") continue # ответ без запроса к нейросети if text == "/clear": messages = [messages[0]] # остаётся только system — личность continue if text == "/start": messages = [] # полный сброс… # …и снова меню личностей — остальное соберёт DeepSeek print("\033[35m" + name + ":\033[0m ", end="", flush=True) # имя цветом
/clear оставляет только messages[0] — первое system-сообщение, то есть личность. А \033[35m включает цвет, и \033[0m обязательно его выключает — иначе весь терминал останется фиолетовым.

Вопрос 1

Рома собрал собеседника, но тот забывает предыдущие фразы: каждый ответ звучит так, будто разговор только начался. Что он забыл добавить в messages?

В список messages нужно добавлять не только вопросы пользователя (роль user), но и прошлые ответы нейросети как роль assistant. Тогда модель видит весь диалог целиком и помнит контекст.

Без assistant-сообщений память теряется, и собеседник отвечает, будто впервые. Кстати, именно этот список и стирает команда /clear.

Вопрос 2

Что на практике меняет параметр stream=True?

Ответ приходит не целиком после паузы, а маленькими кусочками (chunks), и мы печатаем их сразу — текст появляется на глазах, как в ChatGPT.

print(piece, end="", flush=True) выводит кусочки в одну строку и сразу на экран: end="" — без переноса, flush=True — не копить в буфере.

Отладка

Читаем ошибки стрима

ТЕРМИНАЛ
Traceback (most recent call last): File "companion.py", line 21, in <module> piece = chunk.choices[0].delta.content AttributeError: 'NoneType' object has no attribute 'content'

Traceback читаем снизу вверх: последняя строка — что случилось, строкой выше — где. Здесь delta.content оказался None — спасает or "".

Если застряли

  • Скопируйте traceback целиком в чат DeepSeek — попросите объяснить и починить
  • Пустой ключ или .env не найден — проверьте DEEPSEEK_KEY
  • Текст лезет в столбик — забыли end="" в print

Та же привычка, что console.log в «Змейке» и print-метки на уроке 5.

Запомните

Привычки работы с диалогом

01 · Стриминг — по кусочкам
stream=True присылает ответ кусочками (chunks); print(end="", flush=True) печатает их сразу.
02 · system задаёт личность
Первое сообщение с ролью system управляет именем, стилем и характером всех ответов.
03 · assistant — это память
Каждый ответ нейросети дописываем в messages как assistant, иначе диалог забывается. Именно этот список стирает /clear.
04 · Три роли диалога
system — характер, user — ваши слова, assistant — ответы модели.
05 · Ключ — это пароль
Держим в .env, не публикуем, на скриншотах замазываем.
