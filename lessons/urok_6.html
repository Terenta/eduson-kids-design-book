Новый инструмент

Что такое цикл

Цикл повторяет блок кода, чтобы не писать одно и то же руками.

PYTHON
for i in range(3): # повторить ровно 3 раза print("Шаг", i) while hp > 0: # повторять, пока герой жив hp = hp - 10 # каждый ход герой теряет силы
for — повторить N раз или пройти по списку. while — повторять, пока условие истинно. Игра — это цикл: ход за ходом, пока герой не победит или не падёт.

Новый инструмент

Список и словарь

Чтобы игра помнила, что происходит с героем, ей нужно хранилище. В Python их два главных: список и словарь.

PYTHON
inventory = ["факел", "ключ"] # список (list) — набор по порядку state = {"hp": 100, "name": "Артём"} # словарь (dict) — пары ключ-значение print(state["hp"]) # 100 — берём значение по ключу

Список

Набор значений по порядку, в квадратных скобках [ ] через запятую. Удобно для инвентаря.

Словарь

Пары «ключ»: значение в фигурных скобках { }. Значение достаём по ключу: state["hp"].

Память героя

Один словарь — вся память игры

PYTHON
state = {"hp": 100, "inventory": []} state["hp"] -= 20 # получили урон — стало 80 state["inventory"].append("меч") # нашли предмет — он в рюкзаке

Меняем число

-= уменьшает значение. Так у героя убывает здоровье.

Добавляем в список

.append(...) кладёт предмет в конец инвентаря.

Весь прогресс героя — жив ли он и что у него есть — лежит в одном словаре state.

Цикл игры и память сюжета

messages = [{"role": "system", "content": "Ты — мастер игры…"}] while True: # игра идёт по кругу choice = input("Твой ход: ") if choice.lower() == "выход": break # выйти из игры messages.append({"role": "user", "content": choice}) scene = ask_master(messages) # мастер придумывает сцену messages.append({"role": "assistant", "content": scene})

Полезное знание

Параметры и return

Функции общаются двумя способами: параметры — что передаём внутрь, return — что функция отдаёт наружу.

PYTHON
def add(a, b): # a и b — параметры (вход) return a + b # return — результат (выход) result = add(2, 3) # result = 5
Функция — это «чёрный ящик»: дал данные на вход, получил результат на выход. Игрок не знает, что внутри ask_master живёт нейросеть, — он получает готовую сцену.

Разбор кода

Цвет и конец игры

Терминал понимает специальные последовательности: \033[..m переключает цвет, \033[0m возвращает обычный. А проверка HP решает, когда игра окончена:

PYTHON

Цвета видны во встроенном терминале VS Code. В старых системных консолях они могут не появиться — это не ошибка вашего кода.
Проверка · вопрос 1

Изучите ситуацию и ответьте

Вопрос 1

Тимур написал функцию def ask_master(messages):, внутри посчитал ответ в переменную scene, но забыл строку return scene. В main он пишет text = ask_master(messages) — и в text оказывается None (пусто). Почему?

Без return функция ничего не отдаёт наружу, а переменная scene живёт только внутри функции и исчезает, когда функция заканчивается.

Нужно добавить return scene — тогда результат вернётся в text, и сцену будет видно.

Проверка · вопрос 2

Изучите ситуацию и ответьте

Вопрос 2

Лиза хочет, чтобы за каждый ход HP падал на 10: она написала state["hp"] -= 10. Но HP в игре всегда остаётся 100. Оказалось, строку state = {"hp": 100} она поставила внутри цикла while. Где ошибка?

Словарь state создаётся заново на каждом ходу, поэтому память героя каждый раз обнуляется до 100.

Создавать state нужно один раз до цикла, а внутри цикла — только менять его значения.

Отладка

Читаем ошибки игры

Traceback — снизу вверх. Последняя строка — что случилось (KeyError, TypeError…), строкой выше — где.

ТЕРМИНАЛ
Traceback (most recent call last): File "game.py", line 21, in main print(state["hp"]) KeyError: 'hp'

Частые поломки

  • KeyError 'hp' — опечатка в ключе словаря: проверьте кавычки и имя ключа
  • В переменной None — забыли return в функции
  • Мастер отвечает невпопад — проверьте, что messages растёт и отправляется целиком

Непонятную ошибку целиком копируем в чат DeepSeek — привычка из прошлых уроков.
