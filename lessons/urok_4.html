Идея урока

Что такое Canvas

Canvas — это прямоугольная область на странице, на которой JavaScript может рисовать что угодно: линии, прямоугольники, картинки, текст. Удобно для игр и анимаций.

Один тег <canvas> — и у вас есть «лист бумаги», на котором программа рисует кадр за кадром.
Сравнение

Canvas или DOM — когда что

DOM · уроки 1–3

Набор элементов: div, button, li. У каждого свой стиль и события.

Удобно для интерфейсов: формы, списки.

Canvas · урок 4

Нет элементов — только пиксели. Рисуем всё сами кадр за кадром.

Быстро — идеально для игр и анимаций.

Основа рисования

Как рисовать на Canvas

JAVASCRIPT
const canvas = document.getElementById('game'); const ctx = canvas.getContext('2d'); // «кисть» для рисования // Прямоугольник: x, y, ширина, высота ctx.fillStyle = '#10B981'; ctx.fillRect(100, 100, 20, 20); ctx.clearRect(0, 0, 400, 400); // очистить весь холст ctx.fillStyle = 'white'; // текст ctx.font = '20px Inter'; ctx.fillText('Счёт: 5', 10, 30);
ctx (context) — это «кисть». Все рисунки идут через неё.
Система координат

Где у Canvas точка (0, 0)

Точка (0, 0) — в левом верхнем углу.

  • X растёт вправо →
  • Y растёт вниз ↓ — не вверх, как в математике!

Поэтому «движение вниз» — это увеличение Y.

ХОЛСТ
(0,0) ───────────→ X │ ┌──────────────┐ │ │ │ │ │ холст │ ▼ │ 400×400 │ Y └──────────────┘ (400,400)
Сердце любой игры

Что такое игровой цикл

Игра — это цикл, который повторяется много раз в секунду. На каждом «тике» — три действия:

1 · Input

Что нажал игрок? Куда движется змейка?

2 · Update

Двигаем объекты, проверяем столкновения, обновляем счёт.

3 · Render

Стираем холст и рисуем заново: змею, яблоко, счёт.

Фаза 1 · Input

Как игра слышит игрока

Игра не двигает змейку сразу при нажатии. Она запоминает направление, а двигает на следующем тике update.

JAVASCRIPT
let dir = { x: 1, y: 0 }; // едем вправо document.addEventListener('keydown', (e) => { if (e.key === 'ArrowUp') dir = { x: 0, y: -1 }; if (e.key === 'ArrowDown') dir = { x: 0, y: 1 }; if (e.key === 'ArrowLeft') dir = { x: -1, y: 0 }; if (e.key === 'ArrowRight') dir = { x: 1, y: 0 }; });
Input только запоминает намерение. Двигает — фаза Update.
Фаза 2 · Update

Где вся логика игры

В update происходит главное:

  • Двигаем голову змейки на одну клетку в сторону dir
  • Проверяем: голова попала на яблоко? → счёт +1, змейка растёт
  • Проверяем: голова вышла за стену или врезалась в себя? → game over
  • Хвост удаляется, если яблоко не съедено — змейка «ползёт»
Update не рисует ничего. Он только считает новое состояние. Рисование — отдельная фаза.
Фаза 3 · Render

Стереть и нарисовать заново

JAVASCRIPT
function render() { // 1. Стираем весь холст ctx.clearRect(0, 0, 400, 400); // 2. Рисуем каждый сегмент змейки ctx.fillStyle = '#10B981'; for (const seg of snake) { ctx.fillRect(seg.x * 20, seg.y * 20, 19, 19); } // 3. Рисуем яблоко ctx.fillStyle = '#EF4444'; ctx.fillRect(apple.x * 20, apple.y * 20, 19, 19); }
Главное правило: сначала clearRect, иначе старые кадры останутся «следами».
Двигатель цикла

Что такое requestAnimationFrame

Это функция браузера, которая вызывает ваш код перед каждым кадром экрана — обычно 60 раз в секунду.

JAVASCRIPT
function loop() { update(); // посчитать состояние render(); // нарисовать requestAnimationFrame(loop); // запросить следующий кадр } loop(); // запускаем
В отличие от setInterval, она сама подстраивается под устройство: на слабом компьютере кадры реже, но игра остаётся плавной.

Полезный приём

Сетка вместо пикселей

Холст 400×400 пикселей. Клетка — 20 пикселей. Значит сетка — 20×20 клеток.

Думаем в клетках

Змейка на клетке (10, 10). Удобно проверять столкновения.

Рисуем в пикселях

Клетка (10, 10) → пиксели (200, 200): умножаем на 20.

Змейка двигается сама

→ В тот же чат
Доработай script.js: - Змейка — теперь массив сегментов (каждый = клетка с x, y) - Начнём с 3 сегментов в центре - Направление движения — переменная dir {x, y}, по умолчанию {x: 1, y: 0} (вправо) - В update: двигай голову на одну клетку в направлении dir, остальные сегменты следуют (последний удаляется, новый добавляется в начало) - Скорость — 4 кадра в секунду (счётчик внутри requestAnimationFrame) Не трогай стили и HTML. Разбор кода

Как именно ползёт змейка

JAVASCRIPT
function update() { const head = snake[0]; const newHead = { // новая голова на клетку вперёд x: head.x + dir.x, y: head.y + dir.y }; snake.unshift(newHead); // добавили голову в начало snake.pop(); // удалили хвост в конце }
Ключевой приём: мы не двигаем каждый сегмент. Добавляем голову и убираем хвост — на экране это выглядит как ползущая змея.
Правка 2 · управление

Управление стрелками

→ В тот же чат
Добавь обработку клавиатуры: - При нажатии стрелок (ArrowUp, ArrowDown, ArrowLeft, ArrowRight) — меняй направление dir - Запрети разворот на 180°: если змейка двигается вправо, нажатие «влево» не должно сработать - Используй document.addEventListener('keydown', …) - Добавь e.preventDefault(), чтобы стрелки не прокручивали страницу Изменения только в script.js. Верни целиком обновлённый script.js.
Важная тонкость

Почему нельзя развернуться на 180°

✕ Без запрета

Змейка едет вправо. Игрок жмёт «влево». Голова мгновенно врезается в свою же шею — мгновенный проигрыш на ровном месте.

✓ С запретом

Проверяем: новое направление не противоположно текущему. Если противоположно — игнорируем нажатие.

JAVASCRIPT
// нельзя влево, если едем вправо if (newDir.x === -dir.x && newDir.y === -dir.y) return;
Полезное знание

Зачем e.preventDefault()

У стрелок есть стандартное поведение браузера — прокрутка страницы. В игре это мешает: нажал «вниз» — страница уехала.

JAVASCRIPT
document.addEventListener('keydown', (e) => { e.preventDefault(); // отменяем прокрутку страницы // ... меняем направление });
preventDefault() говорит браузеру: «не делай своё стандартное действие, я обработаю сам».

Разбор кода

Случайная позиция яблока

JAVASCRIPT
function randomApple() { let pos; do { pos = { x: Math.floor(Math.random() * 20), // 0..19 y: Math.floor(Math.random() * 20), }; } while (snake.some(s => s.x === pos.x && s.y === pos.y)); return pos; // клетка точно НЕ на змейке }
do...while повторяет генерацию, пока яблоко не окажется на свободной клетке. Иначе яблоко могло бы появиться внутри змейки.
Правка 4 · проигрыш

Game Over при столкновении

→ В тот же чат
Добавь логику проигрыша: - Если голова змейки выходит за границы холста — game over - Если голова попадает в любой сегмент собственного тела — game over - При game over: останови игру, нарисуй полупрозрачный чёрный прямоугольник на весь холст, в центре крупный белый текст «GAME OVER» и снизу «Нажмите Пробел, чтобы начать заново» - При нажатии Пробела — игра начинается с нуля Изменения в script.js. Верни обновлённый script.js.
Разбор кода

Как проверяются столкновения

JAVASCRIPT
const h = snake[0]; // голова // 1. Со стеной — вышли за сетку 0..19 if (h.x < 0 || h.x > 19 || h.y < 0 || h.y > 19) { gameOver(); } // 2. С собой — голова на клетке тела for (let i = 1; i < snake.length; i++) { if (h.x === snake[i].x && h.y === snake[i].y) { gameOver(); } }
Проверку начинаем с i = 1 — голова сама с собой не сталкивается.

Вопрос 1

Змейка двигается слишком быстро — играть невозможно. Какие два способа замедлить движение, не трогая requestAnimationFrame?

1. Счётчик кадров: увеличивать frames++ в цикле и вызывать update() только когда frames % 10 === 0.

2. По времени: запоминать lastTime и обновлять, только если прошло, например, 150 мс.
