Главная идея

Что такое API

Нейросеть можно вызывать не только из чата в браузере. У DeepSeek есть API — дверь для программ: ваш код отправляет вопрос, нейросеть возвращает ответ текстом.

API — это способ, которым программы разговаривают друг с другом. Сегодня ваша программа впервые заговорит с нейросетью напрямую — без браузера и без чата.
Как это работает

Как летит запрос

1 · Запрос

advisor.py собирает сообщение: модель, роль советчика и вопрос пользователя.

2 · Нейросеть

Сервер DeepSeek генерирует ответ — тот же «мозг», что отвечает вам в чате.

3 · Ответ

Программа получает текст и печатает его в терминале.

Всё это — одна функция requests.post и пара строк кода. Их напишет DeepSeek.
Пропуск к нейросети

API-ключ

Ключ — строка вида sk-.... Это пропуск: по нему сервер понимает, кто стучится.

Где взять

Сегодня ключ выдаст преподаватель — вставите его в код и сразу проверите запрос.

Как хранить

Ключ — как пароль: не публиковать, не показывать на скриншотах. Сегодня вставим в константу API_KEY прямо в код — для учёбы можно. Правильному хранению научимся на уроке 8.

Если ключ «утёк» — его перевыпускают: старый перестаёт работать, выдаётся новый.

Разбор кода

Четыре строки, которые ходят к нейросети

PYTHON
url = "https://api.deepseek.com/chat/completions" # адрес DeepSeek headers = {"Authorization": f"Bearer {API_KEY}"} # пропуск: ваш ключ data = { "model": "deepseek-chat", # какая модель отвечает "messages": [ {"role": "system", "content": "Ты дружелюбный советчик…"}, {"role": "user", "content": question}, # вопрос из input ], } response = requests.post(url, headers=headers, json=data) # отправили answer = response.json()["choices"][0]["message"]["content"] # достали текст

headers — пропуск с ключом, data — что отправляем, response — что вернулось. Всё остальное — обёртка.

Разбор кода

Два сообщения — две роли

role: system
Характер и правила советчика. Пишем мы, один раз: «Ты дружелюбный советчик для подростка. Отвечай кратко, 2–3 предложения».
role: user
Живой вопрос пользователя — то, что пришло из input. Меняется при каждом запуске.
Системная роль — это промпт, который живёт внутри программы. Роль, контекст, формат, ограничения — те же 5 техник из урока 2, только теперь в вашем коде.

Промпт — часть программы

PYTHON
characters = { "1": "Ты добрый друг: поддерживай и подбадривай.", "2": "Ты строгий тренер: давай чёткие шаги, без воды.", "3": "Ты весёлый кот: шути и мурлычь, но советуй дельно.", } role = characters.get(choice, characters["1"]) messages = [{"role": "system", "content": role}, ...]

Правка 3 · разговор по кругу

Диалог, а не один ответ

→ В тот же чат
Сейчас советчик отвечает один раз и выключается. Сделай диалог: - Программа спрашивает «{имя}, о чём поговорим?» по кругу - На каждый вопрос — новый ответ нейросети - Цикл крутится, пока пользователь не введёт «выход» - После «выход» — попрощайся по имени Верни весь файл.
Разбор кода

Как работает вечный цикл

PYTHON
while True: # повторять бесконечно question = input(f"{name}, о чём поговорим? ") if question.lower() == "выход": break # выйти из цикла # ...запрос к нейросети и печать ответа...

while True повторяет блок снова и снова, break — выход. Проверка «выход» стоит до запроса, чтобы не тратить обращение к нейросети впустую.

Циклы — главная тема урока 6. Сегодня берём необходимый минимум.
Правка 4 · история диалога

Советчик ведёт дневник

→ В тот же чат
Пусть советчик ведёт дневник: - После каждого ответа дозаписывай в файл history.txt строку вида «[2026-06-12 18:30] Маша: вопрос → ответ нейросети» - Дату и время бери из datetime.now() - Файл открывай через with open('history.txt', 'a', encoding='utf-8') - Если файла нет — он должен создаться сам Верни весь файл.
Разбор кода

with open и режимы файла

PYTHON
from datetime import datetime now = datetime.now().strftime('%Y-%m-%d %H:%M') with open('history.txt', 'a', encoding='utf-8') as f: f.write(f"[{now}] {name}: {question} → {answer}\n")
'r' — читать
Только смотрим содержимое, менять нельзя.
'w' — переписать
Создаёт файл заново и стирает всё старое. Осторожно!
'a' — дописать
Добавляет строки в конец. История копится — нам сюда.

with закрывает файл сам, а encoding='utf-8' бережёт русские буквы от кракозябр.

Правка 5 · цвет в терминале

Раскрашиваем советчика

Советчик работает — теперь наведём красоту. Одна правка ANSI-кодами, и вывод оживёт цветом, без единой сторонней библиотеки.

→ В тот же чат
Раскрась вывод советчика ANSI-кодами, без сторонних библиотек: - Приветствие — фиолетовым - Вопросы пользователю — голубым - Ответ нейросети — зелёным и жирным - Прощание — обычным цветом Заведи в начале файла константы вроде VIOLET = '\033[35m' и RESET = '\033[0m' и подставляй их в print через f-строки. Верни весь обновлённый advisor.py.
Разбор кода

Как терминал понимает цвет

Терминал следит за специальными последовательностями в тексте: \033[35m переключает цвет, \033[0m возвращает обычный. Удобно завести константы в начале файла:

PYTHON
VIOLET = '\033[35m' # фиолетовый CYAN = '\033[96m' # ярко-голубой GREEN = '\033[92m' # ярко-зелёный BOLD = '\033[1m' # жирный RESET = '\033[0m' # вернуть обычный вид print(f"{GREEN}{BOLD}Совет:{RESET}")
Работает во встроенном терминале VS Code. В старых системных консолях цвета могут не появиться — это не ошибка вашего кода.
Проверка · вопрос 1

Изучите ситуацию и ответьте

Вопрос 1

Маша задала советчику один и тот же вопрос два раза подряд — и получила два разных ответа. Серёжа говорит: «У тебя в программе ошибка». Прав ли он?

Нет. Нейросеть не достаёт ответ из готового списка — она генерирует текст заново при каждом запросе, поэтому формулировки отличаются.

Так работает и чат DeepSeek: задайте одинаковый вопрос дважды — получите разные ответы. Нейросеть каждый раз заново формулирует ответ.

Проверка · вопрос 2

Изучите ситуацию и ответьте

Вопрос 2

Алина решила «улучшить» сохранение истории и поменяла режим открытия файла с 'a' на 'w'. Ошибок нет, но после каждого запуска в history.txt остаётся только последняя запись. Почему?

Режим 'w' каждый раз создаёт файл заново и стирает старое содержимое. Для дневника нужен режим 'a' — append, дозапись в конец файла.
