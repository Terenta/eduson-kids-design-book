<div class="eyebrow">Идея урока</div>
<h2>Что такое Canvas</h2>
<p class="lead">Canvas — это <em>прямоугольная область на странице</em>, на которой JavaScript может рисовать что угодно: линии, прямоугольники, картинки, текст. Удобно для игр и анимаций.</p>
<div class="card accent" style="margin-top:24px">

</div>

<h2>Как рисовать на Canvas</h2>
<pre><code class="language-javascript">const canvas = document.getElementById('game');
const ctx = canvas.getContext('2d');   // «кисть» для рисования

// Прямоугольник: x, y, ширина, высота
ctx.fillStyle = '#10B981';

<h2>Что такое игровой цикл</h2>
<p>Игра — это цикл, который повторяется много раз в секунду. На каждом «тике» — три действия:</p>

===== SLIDE 9/38 =====
[NOTES] Детально про input. Игра не реагирует мгновенно — она запоминает намерение и применяет на следующем тике.
[HTML]
<div class="eyebrow">Фаза 1 · Input</div>
<h2>Как игра слышит игрока</h2>
<p>Игра <strong>не двигает змейку сразу</strong> при нажатии. Она запоминает <em>направление</em>, а двигает на следующем тике update.</p>
<pre style="margin-top:14px"><code class="language-javascript">let dir = { x: 1, y: 0 };   // едем вправо

document.addEventListener('keydown', (e) =&gt; {
  if (e.key === 'ArrowUp')    dir = { x: 0, y: -1 };
  if (e.key === 'ArrowDown')  dir = { x: 0, y:  1 };
  if (e.key === 'ArrowLeft')  dir = { x: -1, y: 0 };
  if (e.key === 'ArrowRight') dir = { x: 1, y:  0 };
});</code></pre>
<p style="font-size:14px; margin-top:12px; color:var(--muted)">Input только <em>запоминает</em> намерение. Двигает — фаза Update.</p>

===== SLIDE 10/38 =====
[NOTES] Update — вся логика. Двигаем голову, проверяем яблоко и столкновения.
[HTML]
<div class="eyebrow">Фаза 2 · Update</div>
<h2>Где вся логика игры</h2>
<p>В update происходит главное:</p>
<ul class="check-list" style="margin-top:8px">
  <li>Двигаем голову змейки на одну клетку в сторону <code class="inline">dir</code></li>
  <li>Проверяем: голова попала на яблоко? → счёт +1, змейка растёт</li>
  <li>Проверяем: голова вышла за стену или врезалась в себя? → game over</li>
  <li>Хвост удаляется (если яблоко не съедено) — змейка «ползёт»</li>
</ul>
<div class="card warn" style="margin-top:16px">
  <p style="font-size:14.5px">Update <strong>не рисует</strong> ничего. Он только считает новое состояние. Рисование — отдельная фаза.</p>
</div>

===== SLIDE 11/38 =====
[NOTES] Render — только рисование. Сначала стереть весь холст, потом нарисовать заново.
[HTML]
<div class="eyebrow">Фаза 3 · Render</div>
<h2>Стереть и нарисовать заново</h2>
<pre><code class="language-javascript">function render() {
  // 1. Стираем весь холст
  ctx.clearRect(0, 0, 400, 400);

  // 2. Рисуем каждый сегмент змейки
  ctx.fillStyle = '#10B981';
  for (const seg of snake) {
    ctx.fillRect(seg.x * 20, seg.y * 20, 19, 19);
  }

  // 3. Рисуем яблоко
  ctx.fillStyle = '#EF4444';
  ctx.fillRect(apple.x * 20, apple.y * 20, 19, 19);
}</code></pre>
<p style="font-size:14px; margin-top:12px; color:var(--muted)">Главное правило: <strong>сначала clearRect</strong>, иначе старые кадры останутся «следами».</p>

===== SLIDE 12/38 =====
[NOTES] Что запускает цикл. Браузер сам подбирает частоту ~60 кадров/сек.
[HTML]
<div class="eyebrow">Двигатель цикла</div>
<h2>requestAnimationFrame</h2>
<p>Это функция браузера, которая вызывает ваш код <strong>перед каждым кадром экрана</strong> — обычно 60 раз в секунду.</p>
<pre style="margin-top:14px"><code class="language-javascript">function loop() {
  update();                      // посчитать состояние
  render();                      // нарисовать
  requestAnimationFrame(loop);   // запросить следующий кадр
}
loop();                          // запускаем</code></pre>
<div class="card accent" style="margin-top:14px">
  <p style="font-size:14.5px">В отличие от <code class="inline">setInterval</code>, она сама подстраивается под устройство: на слабом компьютере кадры реже, но игра остаётся плавной.</p>
</div>
