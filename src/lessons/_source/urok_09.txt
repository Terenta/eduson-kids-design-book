from telegram import Update
from telegram.ext import (Application,
    MessageHandler, filters, ContextTypes)

TOKEN = "123456:ABC-DEF..."   # пока прямо здесь

async def on_message(update: Update,
        context: ContextTypes.DEFAULT_TYPE):
    text = update.message.text
    await update.message.reply_text(text)

app = Application.builder().token(TOKEN).build()
app.add_handler(MessageHandler(
    filters.TEXT &amp; ~filters.COMMAND, on_message))
app.run_polling()   # бот спрашивает Telegram</code></pre>

===== SLIDE 25/38 =====
[NOTES] Зафиксируйте два слова: async у обработчика и await перед reply_text. Идея простая: пока бот ждёт ответ от DeepSeek для одного человека, он не блокирует обработку других сообщений. Без await ответ не дойдёт — это частая ошибка.
[HTML]
<div class="eyebrow">Полезное знание</div>
<h2>Два слова: async и await</h2>
<div class="vs-grid">
  <div class="vs-col minimal">
    <h4>Обычная функция</h4>
    <p>Бот занят одним человеком — остальные ждут в очереди, пока придёт ответ.</p>
    <p class="note">Один медленный запрос блокирует остальных.</p>
  </div>
  <div class="vs-col detailed">
    <h4>async def и await</h4>
    <p>Пока бот ждёт ответ DeepSeek для одного — он принимает сообщения от других.</p>
    <p class="note"><code class="inline">await</code> перед <code class="inline">reply_text</code> обязателен.</p>
  </div>
</div>
<p style="font-size:14px; margin-top:14px; color:var(--muted)">Обработчик — это <code class="inline">async def</code>, а перед отправкой ответа стоит <code class="inline">await</code>. Так бот не блокируется на одном пользователе и остаётся доступным для всех.</p>

<h2>Почему память — словарь по user_id</h2>
<p>Бот один, а собеседников много. У каждого своя история, поэтому храним её в словаре по <code class="inline">user_id</code>. История — это <em>список сообщений с ролями</em>, тот самый формат, что уходит в DeepSeek на уроках 5-8:</p>
<pre style="margin-top:14px"><code class="language-python">history = {}                       # user_id -&gt; список сообщений с ролями

async def on_message(update: Update, context: ContextTypes.DEFAULT_TYPE):
    user_id = update.message.from_user.id
    msgs = history.setdefault(user_id, [])          # своя история для каждого
    msgs.append({"role": "user", "content": text})  # запомнили вопрос человека
    answer = ask_deepseek(msgs)                      # весь список сообщений — в DeepSeek
    msgs.append({"role": "assistant", "content": answer})  # запомнили и ответ бота</code></pre>
<div class="card accent" style="margin-top:14px">
  <p style="font-size:14.5px">В <code class="inline">msgs</code> лежат словари <code class="inline">{"role": "user"...}</code> и <code class="inline">{"role": "assistant"...}</code> по очереди — именно этот список нейросеть читает как весь разговор. Одна общая переменная не подошла бы: сообщения Вани и Насти смешались бы. Ключ словаря <code class="inline">user_id</code> разделяет истории — у каждого <strong>свой контекст</strong>.</p>
</div>
