<h2>Что такое DOM</h2>
<p class="lead">DOM (Document Object Model) — это <em>модель страницы в памяти браузера</em>. Каждый тег HTML превращается в узел, к которому можно обратиться из JavaScript.</p>

  </div>
  <div class="card accent">

    <p style="font-size:14.5px">То, во что браузер превращает HTML, чтобы JS мог это менять «на лету».</p>
  </div>
</div>

===== SLIDE 9/38 =====
[NOTES] Два главных способа: getElementById и querySelector. Покажите оба.
[HTML]
<div class="eyebrow">Шаг 1 работы с DOM</div>
<h2>Как JavaScript находит элемент</h2>
<div class="grid cols-2">
  <div>
    <h3>По id</h3>
<pre><code class="language-javascript">// HTML: &lt;button id="add"&gt;

  </div>

<div class="card warn" style="margin-top:18px">
  <p style="font-size:14.5px"><strong>Правило:</strong> <code class="inline">querySelector</code> возвращает <em>один</em> элемент (первый совпавший). <code class="inline">querySelectorAll</code> — <em>список</em> всех совпавших.</p>
</div>

===== SLIDE 10/38 =====
[NOTES] Три главных операции: поменять текст, добавить/убрать класс, создать новый элемент.
[HTML]
<div class="eyebrow">Шаг 2 работы с DOM</div>
<h2>Как JavaScript меняет элемент</h2>
<pre><code class="language-javascript">// 1. Поменять текст
document.querySelector('h1').textContent = 'Привет!';

// 2. Добавить / убрать CSS-класс
element.classList.add('done');
element.classList.remove('done');
element.classList.toggle('done');   // есть → убрать, нет → добавить

// 3. Создать новый элемент и вставить
const li = document.createElement('li');
li.textContent = 'Новая задача';
document.querySelector('ul').appendChild(li);

// 4. Удалить элемент
li.remove();</code></pre>
<p style="font-size:14px; margin-top:14px; color:var(--muted)">Этих 4 операций хватит, чтобы собрать список задач целиком.</p>

===== SLIDE 11/38 =====
[NOTES] Событие — это «сигнал», который браузер отправляет, когда что-то произошло. JS только подписывается на нужные.
[HTML]
<div class="eyebrow">Сигналы от пользователя</div>
<h2>Что такое событие</h2>
<p class="lead">Событие (event) — это <em>сигнал от браузера</em>: что-то произошло. Клик, ввод символа, отметка чекбокса, нажатие клавиши. JavaScript не «следит» постоянно — он <strong>подписывается</strong> на нужные события и ждёт.</p>
<div class="card accent" style="margin-top:24px">
  <p style="font-size:16px; color:#E2E8F0">Аналогия: вы не стоите весь день у двери. Вы вешаете <em>звонок</em>. Когда придёт гость — звонок сработает. <code class="inline">addEventListener</code> — это и есть «повесить звонок».</p>
</div>

[NOTES] Таблица главных событий. Сегодня используем click, input, change.
[HTML]
<div class="eyebrow">Справочник</div>
<h2>Главные виды событий</h2>
<div class="kv-row"><div class="k">click</div><div class="v">Клик мышью по элементу. Кнопки, ссылки, любые блоки.</div></div>

<div class="kv-row"><div class="k">change</div><div class="v">Значение изменилось и поле потеряло фокус. Чекбоксы, выпадающие списки.</div></div>
<div class="kv-row"><div class="k">submit</div><div class="v">Отправка формы (нажатие Enter или кнопки в форме).</div></div>
<div class="kv-row"><div class="k">keydown</div><div class="v">Нажата клавиша на клавиатуре. Пригодится на уроке 4 для игры.</div></div>
<div class="kv-row"><div class="k">mouseover</div><div class="v">Курсор навёлся на элемент. Часто заменяют CSS-эффектом <code class="inline">:hover</code>.</div></div>

[NOTES] localStorage хранит строки. Массив объектов — через JSON.stringify туда и JSON.parse обратно.
[HTML]
<div class="eyebrow">Ключевой приём</div>
<h2>Массив → строка → массив</h2>
<p>localStorage хранит только <strong>строки</strong>. Массив задач превращаем в строку и обратно:</p>

  { text: 'Купить хлеб', done: false },

];

localStorage.setItem('tasks', JSON.stringify(tasks));

// Прочитать: строка JSON → массив

<div class="card warn" style="margin-top:14px">
